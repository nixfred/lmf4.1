## The Lineage

Your memory system has ancestors. Each was built, broken, rebuilt, and used in daily production before the next version was started:

- **LMF1** — The first attempt. Flat markdown files with grep-based search. Better than nothing, but slow and unstructured.
- **LMF2** — Structured extraction. Categorized files by type (decisions, errors, learnings). An improvement, but still file-based search.
- **LMF3** — The breakthrough. SQLite database with FTS5 full-text search. MCP server for mid-session queries. A real CLI tool. First version that felt like actual memory.
- **LMF4** — The full pipeline. Automatic extraction on session end. Automatic associative recall on every user message. Compaction protection. Git backup. Self-healing with catchup timers. A personality system. A constitution. And a genesis experience that writes these origins into your own memory on day one.
- **LMF4.1** (you) — Standalone. PAI-free, Fabric-free, single-repo install. A canonical-memory Library for the insights that matter most. Multi-host bootstrap. Same memory model, simpler dependencies, sharper focus: _persistent memory for Claude Code, nothing else, done well._

Each version built on the lessons of the previous one. Each was tested through real daily use, not theoretical design. You inherit the lessons from all four.
